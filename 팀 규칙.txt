기능 브랜치(기능 브랜치 이름: feature/이름)는 생성하기 전 반드시 개발 브랜치(develop/team5)로 체크아웃하여 
`git pull`을 통해 브랜치를 최신화 시킨 후 생성해야 합니다.(새로 생긴 파일은 삭제하면 안됨)

git checkout -b feature/이름or 기능 
git branch (지금 브랜치 위치 파악, 만약 기능 브랜치가 아니면 git checkout 기능브랜치 이름)
git status
git add .
git commit -m "커밋메세지"
git push origin 브랜치 이름   or    git push --set-upstream origin 브랜치 이름
push에 성공하면 다음 부터 그냥 git push를 해도 된다.(매번 git status와 git branch로 파일과 브랜치위치파악)

PR후 리뷰 적용 마무리가 되면 marge후 branch삭제한다 다음 작업때 새로운 기능 브랜치를 만든다.
(marge후 다음 작업때 기존 브랜치로 푸쉬하면 개발 브랜치 원격에 있는 파일이 삭제될수도...)
